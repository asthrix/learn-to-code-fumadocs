## What Are Props?

**Props** (short for "properties") are how you pass data from a parent component
to a child component. Think of them like function arguments:

```jsx title="Props vs Function Parameters"
// Function with parameters
function greet(name) {
   return `Hello, ${name}!`;
}

// Component with props
function Greeting({ name }) {
   return <h1>Hello, {name}!</h1>;
}

// Usage
<Greeting name='Sarah' />;
```

Props make components **reusable** - same component, different data!

Create a new file in the `components` folder:

```jsx title="src/components/PropertyCard.jsx"
         </div>
      </div>
   );

export default PropertyCard;
```

## Understanding Props Destructuring

Let's break down the component signature:

```jsx title="Props Destructuring"
function PropertyCard({ title, location, price, rating, image }) {
   // ...
}
```

This is **destructuring** the props object. It's equivalent to:

```jsx title="Without Destructuring"
function PropertyCard(props) {
   const title = props.title;
   const location = props.location;
   const price = props.price;
   // ... and so on

   // Then use: {title}, {location}, etc.
}
```

**Destructuring is cleaner!** We extract the values we need directly in the
parameters.

## How Props Work

When you write:

```jsx title="Passing Props"
<PropertyCard
   title='Cozy Beach House'
   location='Malibu, California'
   price={250}
/>
```

React creates a props object:

```javascript title="Props Object"
{
  title: "Cozy Beach House",
  location: "Malibu, California",
  price: 250
}
```

And passes it to the PropertyCard function!

## Props Syntax Rules

<Steps>
   <Step>
      ### String Props
      
      Use quotes for strings:
      
      ```jsx title="String Props"
      <PropertyCard title='Beach House' />
      ```
   </Step>

   <Step>
      ### Number Props
      
      Use curly braces for numbers:
      
      ```jsx title="Number Props"
      <PropertyCard price={250} />
      ```
   </Step>

   <Step>
      ### Variable Props
      
      Use curly braces for variables:
      
      ```jsx title="Variable Props"
      const propertyTitle = "Beach House";
      <PropertyCard title={propertyTitle} />
      ```
   </Step>

   <Step>
      ### Object Properties as Props
      
      Access object properties:
      
      ```jsx title="Object Property Props"
      <PropertyCard title={properties[0].title} />
      ```
   </Step>
</Steps>

## Why This is Powerful

The same `PropertyCard` component can display different properties:

```jsx title="Component Reusability"
// Same component, different data!
<PropertyCard title="Beach House" price={250} />
<PropertyCard title="City Loft" price={180} />
<PropertyCard title="Mountain Cabin" price={320} />
```

**Write once, use everywhere!** This is the core principle of component-based
architecture.

## Props vs. State

<Accordions type='single'>
   <Accordion id='props' title='Props (What We Just Learned)' value='props'>
      - **Passed from parent** to child
      - **Read-only** (child can't modify them)
      - Used to configure a component
      - Like function parameters
      
      ```jsx title="Props Example"
      <PropertyCard price={250} /> // Parent passes
      // PropertyCard receives and displays
      ```
   </Accordion>

   <Accordion id='state' title='State (Coming in Module 2)' value='state'>
      - **Managed within** a component
      - **Can be modified** by the component
      - Causes re-renders when changed
      - Used for interactive data
      
      ```jsx title="State Example (Preview)"
      const [count, setCount] = useState(0);
      // Component owns and can change this
      ```
   </Accordion>
</Accordions>

## Common Props Mistakes

<Accordions type="single">
  <Accordion id="wrong-curly" title="Forgetting Curly Braces for Numbers" value="wrong-curly">
    ```jsx title="Number Prop Syntax"
    // ❌ Wrong: Number as string
    <PropertyCard price="250" />
    // PropertyCard receives: "250" (string)
    
    // ✅ Correct: Number as number
    <PropertyCard price={250} />
    // PropertyCard receives: 250 (number)
    ```
    
    **Why it matters:** Math operations won't work on strings!
  </Accordion>
  
  <Accordion id="typo" title="Mismatched Prop Names" value="typo">
    ```jsx title="Prop Name Mismatch"
    // In PropertyCard:
    function PropertyCard({ title, location }) {
      return <h3>{title}</h3>;
    }
    
    // ❌ Wrong prop name
    <PropertyCard name="Beach House" />
    // {title} will be undefined!
    
    // ✅ Correct prop name
    <PropertyCard title="Beach House" />
    ```
  </Accordion>
  
  <Accordion id="missing-import" title="Forgetting to Import" value="missing-import">
    ```jsx title="Import Requirements"
    // ❌ Will cause error: PropertyCard is not defined
    function HomePage() {
      return <PropertyCard title="House" />;
    }
    
    // ✅ Always import first
    import PropertyCard from '../components/PropertyCard';
    
    function HomePage() {
      return <PropertyCard title="House" />;
    }
    ```
  </Accordion>
</Accordions>
