Let's integrate the Navbar into our application so it appears on every page!

## Current App Structure

Currently, our `App.jsx` looks like this:

```jsx title="src/App.jsx"
import { Provider } from 'react-redux';
import { store } from './state/store';
import { Router } from './components/Router';

export function App() {
  return (
    <Provider store={store}>
      <Router />
    </Provider>
  );
}
```

The `Router` component contains all our routes. We need to add the Navbar **above** the Router so it appears on every page.

## Where to Add the Navbar?

We have two options:

<Tabs items={['In App.jsx (Recommended)', 'In Router.jsx (Alternative)']}>

<Tab value="In App.jsx (Recommended)">

**Add Navbar in `App.jsx`:**

```jsx
<Provider store={store}>
  <Navbar />  {/* Shows on all pages */}
  <Router />
</Provider>
```

**Pros:**
- âœ… Navbar outside routing logic
- âœ… Cleaner separation of concerns
- âœ… Easier to add other global components

**This is what we'll do!**

</Tab>

<Tab value="In Router.jsx (Alternative)">

**Add Navbar inside `Router.jsx`:**

```jsx
<BrowserRouter>
  <Navbar />  {/* Inside router */}
  <Routes>
    <Route path="/" element={<HomePage />} />
    {/* ... */}
  </Routes>
</BrowserRouter>
```

**Pros:**
- âœ… All layout in one file

**Cons:**
- âŒ Mixes routing with layout
- âŒ Less flexible

</Tab>

- Proper semantic HTML (`<main>`)

## Improving the Layout

Let's add a container and proper spacing:

```jsx title="src/App.jsx"
import { Provider } from 'react-redux';
import { store } from './state/store';
import { Router } from './components/Router';
import Navbar from './components/Navbar';

export function App() {
  return (
    <Provider store={store}>
      <div className="min-h-screen flex flex-col">
        <Navbar />
        <main className="flex-1">
          <Router />
        </main>
      </div>
    </Provider>
  );
}
```

**What changed?**

<Accordions>

<Accordion title="min-h-screen">

```jsx
<div className="min-h-screen">
```

**Ensures full viewport height:**

- `min-h-screen` = `min-height: 100vh`
- Content will fill at least the full screen height
- If content is longer, page scrolls normally

**Why?**
- Prevents short pages from having white space at bottom
- Professional look
- Footer stays at bottom on short pages

</Accordion>

<Accordion title="flex flex-col">

```jsx
<div className="min-h-screen flex flex-col">
```

**Creates vertical flex container:**

- `flex` - Enable flexbox
- `flex-col` - Stack children vertically

**Children:** Navbar â†’ main â†’ (footer)

</Accordion>

<Accordion title="flex-1 on main">

```jsx
<main className="flex-1">
```

**Makes content fill available space:**

- `flex-1` = `flex: 1 1 0%`
- Takes all remaining space
- Pushes footer to bottom
