# Validating Module 6

## Learning Objectives

By the end of this lesson, you will:

-  [ ] Execute automated tests for Redux and Zustand logic.
-  [ ] Inspect React DevTools profiler to ensure renders stay predictable.
-  [ ] Validate optimistic updates, persistence, and rollback flows.
-  [ ] Capture metrics and TODOs for future state work.

## Test Suite

-  Run reducer and selector unit tests (`npm run test -- state`).
-  Add integration tests for thunks using mocked API clients.
-  Create Jest or Vitest tests for Zustand stores (set/get, persistence).
-  Ensure Context providers have smoke tests to prevent regression.

---

## Profiling Checklist

1. Record dashboard interactions before/after migration.
2. Monitor `selectTasksByProject` memoisation; confirm minimal recomputes.
3. Verify Zustand selectors only re-compute when dependencies change.
4. Inspect thunk duration metrics via analytics (if wired).

---

## Documentation Updates

-  Log metrics and validation results in `notes/module-6-state-plan.md`.
-  Update `docs/state-architecture.md` with baseline data, including Zustand
   persistence notes.
-  File issues for remaining improvements (e.g., RTK Query adoption, store
   hydration helpers).
