## The Route Component

The `<Route>` component creates a mapping between a URL and a component:

```jsx title="Route Syntax"
<Route path="/some-url" element={<SomeComponent />} />
```

**Two required props:**

1. **`path`** - The URL pattern to match
2. **`element`** - The JSX component to render

<Callout type="info" title="Element Prop">

Notice `element` receives JSX (`<Component />`), not a reference (`Component`). This changed in React Router v6!

**v6 (Current):** `element={<HomePage />}`  
**v5 (Old):** `component={HomePage}`

</Callout>

## Understanding Paths

The `path` prop defines what URL triggers this route:

<Tabs items={['Index Route', 'Static Route', 'Dynamic Route']}>

<Tab value="Index Route">

**Path:** `"/"`

**Matches:** The root/homepage URL

```jsx
<Route path="/" element={<HomePage />} />
```

**Examples:**
- ‚úÖ `http://localhost:5173/`
- ‚úÖ `http://example.com/`
- ‚ùå `http://localhost:5173/about`

</Tab>

<Tab value="Static Route">

**Path:** Any fixed string

**Matches:** Exact URL

```jsx
<Route path="/about" element={<AboutPage />} />
<Route path="/contact" element={<ContactPage />} />
<Route path="/pricing" element={<PricingPage />} />
```

**Examples:**
- ‚úÖ `/about` ‚Üí AboutPage
- ‚úÖ `/contact` ‚Üí ContactPage
- ‚ùå `/about/team` ‚Üí No match (unless nested)

</Tab>

<Tab value="Dynamic Route">

**Path:** Contains parameters with `:`

**Matches:** Variable segments

```jsx
<Route path="/listing/:id" element={<DetailsPage />} />
<Route path="/user/:username" element={<ProfilePage />} />
```

**Examples:**
- ‚úÖ `/listing/1` ‚Üí DetailsPage (id="1")
- ‚úÖ `/listing/42` ‚Üí DetailsPage (id="42")
- ‚úÖ `/user/john` ‚Üí ProfilePage (username="john")

We'll cover this in Lesson 5!

</Tab>

</Tabs>

```jsx title="src/components/Router.jsx" {2}
        {/* Routes will be defined here */}

```

</Step>

<Step>

#### Define the Route

Add the Route inside `<Routes>`:

```jsx title="src/components/Router.jsx" {9}
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import HomePage from '../pages/HomePage';

export default function Router() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<HomePage />} />
      </Routes>
    </BrowserRouter>
  );
}
```

**What this does:**

- When URL is exactly `/`, render `<HomePage />`
- No other URLs will match yet

## How Route Matching Works

When you navigate to a URL, React Router:

1. **Looks at the current URL** - Example: `/`
2. **Checks each `<Route>` in order** - Finds `path="/"`
3. **Finds the first match** - Matches our HomePage route
4. **Renders that route's element** - Renders `<HomePage />`

<Callout type="info" title="Exact Matching">

In React Router v6, paths match exactly by default. You don't need an `exact` prop like in v5!

**v6:** `<Route path="/" element={<Home />} />`  
**v5:** `<Route exact path="/" component={Home} />`

</Callout>

## Route Order and Priority

<Tabs items={['Specific to General', 'Common Pitfall']}>

<Tab value="Specific to General">

**Best practice:** Put more specific routes before general ones

```jsx title="Good Order"
<Routes>
  <Route path="/" element={<HomePage />} />
  <Route path="/listing/:id" element={<DetailsPage />} />
  <Route path="/about" element={<AboutPage />} />
  <Route path="*" element={<NotFoundPage />} />
</Routes>
```

**Why:** React Router picks the first match, and `*` matches everything!

</Tab>

<Tab value="Common Pitfall">

**Don't do this:**

```jsx title="Bad Order"
<Routes>
  <Route path="*" element={<NotFoundPage />} />
  <Route path="/" element={<HomePage />} />
</Routes>
```

**Problem:** `*` catches ALL routes, including `/`. HomePage never shows!

**Rule:** Always put catch-all routes (`*`) last.

</Tab>

</Tabs>

## Multiple Routes Example

Here's what your Router will look like by the end of this module:

```jsx title="src/components/Router.jsx (Preview)"
// [!code word:Route]
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import HomePage from '../pages/HomePage';
import ListingDetailsPage from '../pages/ListingDetailsPage';
import NotFoundPage from '../pages/NotFoundPage';

export default function Router() {
  return (
    <BrowserRouter>
      <Routes>
        {/* Index route */}
        <Route path="/" element={<HomePage />} />
        
        {/* Dynamic route */}
        <Route path="/listing/:id" element={<ListingDetailsPage />} />
        
        {/* Catch-all (404) */}
        <Route path="*" element={<NotFoundPage />} />
      </Routes>
    </BrowserRouter>
  );
}
```

## Understanding the Element Prop

The `element` prop can accept any JSX:

<Accordions>

<Accordion title="Simple Component">

Most common - render a component:

```jsx
<Route path="/" element={<HomePage />} />
```

</Accordion>

<Accordion title="Component with Props">

Pass props to the routed component:

```jsx
<Route 
  path="/dashboard" 
  element={<Dashboard user={currentUser} />} 
/>
```

</Accordion>

<Accordion title="Wrapped Component">

Wrap with providers or layouts:

```jsx
<Route 
  path="/admin" 
  element={
    <AdminLayout>
      <AdminDashboard />
    </AdminLayout>
  } 
/>
```

</Accordion>

<Accordion title="Conditional Rendering">

Render different components based on state:

```jsx
<Route 
  path="/profile" 
  element={isLoggedIn ? <Profile /> : <LoginPage />} 
/>
```

</Accordion>

</Accordions>

## Route vs Routes

Don't confuse these two components!

<Tabs items={['Routes (Container)', 'Route (Individual)']}>

<Tab value="Routes (Container)">

**`<Routes>`** - Plural, wraps multiple routes

**Purpose:** Container for all `<Route>` components

**Usage:**

```jsx
<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/about" element={<About />} />
</Routes>
```

**Key features:**
- Matches the current URL
- Renders the first matching route
- Only one route renders at a time

</Tab>

<Tab value="Route (Individual)">

**`<Route>`** - Singular, one route definition

**Purpose:** Defines a single URL ‚Üí Component mapping

**Usage:**

```jsx
<Route path="/home" element={<HomePage />} />
```

**Key features:**
- Must be inside `<Routes>`
- Has `path` and `element` props
- Can have URL parameters

</Tab>

</Tabs>

## Common Mistakes

<Accordions>

<Accordion title="Forgetting to Import Component">

```jsx
// ‚ùå Error: HomePage is not defined
<Route path="/" element={<HomePage />} />

// ‚úÖ Import it first
import HomePage from '../pages/HomePage';
<Route path="/" element={<HomePage />} />
```

</Accordion>

<Accordion title="Wrong Element Syntax">

```jsx
// ‚ùå Don't pass component reference
<Route path="/" element={HomePage} />

// ‚úÖ Pass JSX element
<Route path="/" element={<HomePage />} />
```

</Accordion>

<Accordion title="Route Outside Routes">

```jsx
// ‚ùå Route must be inside Routes
<BrowserRouter>
  <Route path="/" element={<HomePage />} />
</BrowserRouter>

// ‚úÖ Wrap with Routes
<BrowserRouter>
  <Routes>
    <Route path="/" element={<HomePage />} />
  </Routes>
</BrowserRouter>
```

</Accordion>

</Accordions>

## Debugging Routes

If your route isn't working:

<Steps>

<Step>

#### Check the Console

Look for errors like:

- `Cannot read property 'path' of undefined` ‚Üí Route syntax error
- `Component is not defined` ‚Üí Missing import
- `useRoutes() may be used only...` ‚Üí Missing BrowserRouter

</Step>

<Step>

#### Verify Path Matches URL

```jsx
// Route
<Route path="/home" element={<HomePage />} />

// URL must exactly match
http://localhost:5173/home  ‚úÖ
http://localhost:5173/      ‚ùå (doesn't match)
```

</Step>

<Step>

#### Check Route Order

Make sure specific routes come before general ones:

```jsx
<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/about" element={<About />} />
  <Route path="*" element={<NotFound />} /> {/* Last! */}
</Routes>
```

</Step>

</Steps>
