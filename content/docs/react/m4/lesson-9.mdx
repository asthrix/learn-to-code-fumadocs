Right now, users have to manually type URLs to navigate between pages. Let's fix that by adding clickable links to listing cards!

React Router provides the `Link` component for navigation without page reloads - it's faster and preserves application state.

## Link vs Anchor Tag

<Tabs items={['Link Component', 'Anchor Tag', 'Comparison']}>

<Tab value="Link Component">

**React Router Link:**

```jsx
import { Link } from 'react-router-dom';

<Link to="/listing/42">View Listing</Link>
```

**How it works:**
- Prevents page reload
- Uses History API
- Preserves React state
- Updates URL
- Fast navigation

**Perfect for internal navigation!**

</Tab>

<Tab value="Anchor Tag">

**Regular HTML `<a>` tag:**

```jsx
<a href="/listing/42">View Listing</a>
```

**How it works:**
- Full page reload
- Loses React state
- Re-downloads JavaScript
- Re-initializes app
- Slower

**Use for external links only!**

</Tab>

<Tab value="Comparison">

| Feature | Link | `<a>` tag |
|---------|------|-----------|
| Page reload | ‚ùå No | ‚úÖ Yes |
| Speed | ‚ö° Fast | üêå Slow |
| State preserved | ‚úÖ Yes | ‚ùå No |
| React Router | ‚úÖ Yes | ‚ùå No |
| External sites | ‚ùå No | ‚úÖ Yes |

**Rule of thumb:**
- Internal navigation ‚Üí `<Link>`
- External sites ‚Üí `<a>`

</Tab>

</Tabs>

## Building Dynamic URLs

The key part is the `to` prop with template literal:

```jsx
<Link to={`/listing/${listing.id}`}>
```

**How it works:**

```jsx
// If listing.id = 1
to={`/listing/${listing.id}`}  // ‚Üí "/listing/1"

// If listing.id = 42
to={`/listing/${listing.id}`}  // ‚Üí "/listing/42"

// If listing.id = 999
to={`/listing/${listing.id}`}  // ‚Üí "/listing/999"
```

**Each card links to its own details page!**

## Link vs useNavigate

<Tabs items={['Link Component', 'useNavigate Hook', 'When to Use']}>

<Tab value="Link Component">

**For user-initiated clicks:**

```jsx
<Link to="/listing/42">
  <button>View Details</button>
</Link>
```

**Best for:**
- Clickable elements
- Cards, buttons, text links
- Standard navigation
- Keyboard accessible

</Tab>

<Tab value="useNavigate Hook">

**For programmatic navigation:**

```jsx
import { useNavigate } from 'react-router-dom';

function Component() {
  const navigate = useNavigate();

  const handleSubmit = () => {
    // Do something
    navigate('/success');
  };

  return <button onClick={handleSubmit}>Submit</button>;
}
```

**Best for:**
- After form submissions
- Conditional redirects
- After API calls
- Timeout-based navigation

</Tab>

<Tab value="When to Use">

**Use Link when:**
- User clicks to navigate
- Need crawlable links (SEO)
- Standard navigation patterns
- Want right-click ‚Üí open in new tab

**Use useNavigate when:**
- After form submission
- Conditional logic (if/else)
- Redirect after login
- Navigate after timer

**Example:**

```jsx
// ‚úÖ Link - user click
<Link to="/profile">Go to Profile</Link>

// ‚úÖ useNavigate - after action
function LoginForm() {
  const navigate = useNavigate();

  const handleLogin = async () => {
    await login();
    navigate('/dashboard'); // Redirect after login
  };

  return <button onClick={handleLogin}>Login</button>;
}
```

</Tab>

</Tabs>

## Debugging Link Issues

<Accordions>

<Accordion title="Link doesn't navigate">

**Check:**

1. Is Link imported from `react-router-dom`?

```jsx
import { Link } from 'react-router-dom'; // ‚úÖ
```

2. Is `to` prop spelled correctly?

```jsx
<Link to="/listing/42"> // ‚úÖ
<Link too="/listing/42"> // ‚ùå Typo
```

3. Is component inside Router?

```jsx
<BrowserRouter>
  <Routes>
    <Route path="/" element={<HomePage />} />
  </Routes>
</BrowserRouter>
```

</Accordion>

<Accordion title="Page reloads instead of client-side navigation">

**Cause:** Using `<a>` instead of `<Link>`

```jsx
// ‚ùå Full reload
<a href="/listing/42">View</a>

// ‚úÖ Client-side navigation
<Link to="/listing/42">View</Link>
```

</Accordion>

<Accordion title="404 on Link click">

**Cause:** Route doesn't exist

```jsx
// Link
<Link to="/listing/42">

// But no matching route!
<Routes>
  <Route path="/" element={<HomePage />} />
  {/* Missing /listing/:id route! */}
</Routes>

// Fix: Add route
<Route path="/listing/:id" element={<DetailsPage />} />
```

</Accordion>
