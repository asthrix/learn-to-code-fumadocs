# Creating the ListingDetailsPage

Time to build a details page! When users click on a listing card, they'll navigate to a dedicated page showing full information about that property. Let's create the `ListingDetailsPage` component.

## What You'll Learn

- How to create new page components
- Structuring a details page
- Preparing for dynamic routing
- Best practices for page-level components

## Why We Need a Details Page

Right now, your `ListingCard` shows limited information:
- Title
- Location
- Price per night
- Rating and reviews
- Small image carousel

But users need to see much more before booking:
- 📸 **Full image gallery** - All property photos
- 📝 **Complete description** - Detailed property info
- 🛏️ **Amenities** - WiFi, kitchen, parking, etc.
- 🏠 **Property type** - Entire place, private room, shared room
- 👥 **Host information** - Name, profile, reviews
- 📅 **Availability calendar** - Check-in/check-out dates
- 💰 **Pricing breakdown** - Nightly rate, cleaning fee, service fee
- ⭐ **Detailed reviews** - Guest feedback and ratings
- 📍 **Map location** - Where the property is located

A dedicated details page provides space for all this information!

## Page vs Component

Before we build, let's understand the difference:

<Tabs items={['Page Component', 'Regular Component']}>

<Tab value="Page Component">

**Page components** represent entire routes/views:

**Characteristics:**
- Live in `src/pages/` folder
- Represent full screens/routes
- Handle their own data fetching
- Usually named with "Page" suffix
- Connected to routes in Router

**Examples:**

```
HomePage.jsx        -> /
ListingDetailsPage.jsx -> /listing/:id
ProfilePage.jsx     -> /profile
NotFoundPage.jsx    -> /404
```

**Structure:**

```jsx
export default function ListingDetailsPage() {
  // Data fetching
  // State management
  // Layout
  
  return (
    <div className="page-container">
      {/* Page content with multiple components */}
    </div>
  );
}
```

</Tab>

<Tab value="Regular Component">

**Regular components** are reusable pieces:

**Characteristics:**
- Live in `src/components/` folder
- Represent UI pieces
- Receive data via props
- Named after what they display
- Used within pages

**Examples:**

```
ListingCard.jsx
ImageCarousel.jsx
Spinner.jsx
ErrorMessage.jsx
```

**Structure:**

```jsx
export default function ListingCard({ listing }) {
  // Just display logic
  
  return (
    <div className="card">
      {/* Card content */}
    </div>
  );
}
```

</Tab>

</Tabs>

### Understanding the Layout Structure

Let's break down the JSX structure:

<Accordions>

<Accordion title="Container Classes">

```jsx
<div className="container mx-auto px-4 py-8">
```

**What each class does:**
- `container` - Max-width container (responsive)
- `mx-auto` - Center horizontally with margin auto
- `px-4` - Padding left/right (1rem = 16px)
- `py-8` - Padding top/bottom (2rem = 32px)

This creates a centered, padded content area that works on all screen sizes.

</Accordion>

<Accordion title="Heading Styles">

```jsx
<h1 className="text-3xl font-bold text-gray-900 mb-4">
```

**What each class does:**
- `text-3xl` - Large font size (1.875rem)
- `font-bold` - Bold font weight (700)
- `text-gray-900` - Very dark gray text
- `mb-4` - Margin bottom (1rem = 16px)

Standard heading styles for page titles.

</Accordion>

<Accordion title="Paragraph Styles">

```jsx
<p className="text-gray-600">
```

**What it does:**
- `text-gray-600` - Medium gray text (for secondary content)

Body text uses lighter colors to create visual hierarchy.

</Accordion>

**Purpose:** Reserved space for image carousel

**Styling:**

- `flex items-center justify-center` - Center the text
- `rounded-lg` - Rounded corners

This makes it clear where images will go!

</Accordion>

<Accordion title="Grid Layout">

```jsx
<div className="grid grid-cols-1 lg:grid-cols-3 gap-8">
  {/* Left: 2 columns */}
  <div className="lg:col-span-2">...</div>
  
  {/* Right: 1 column */}
  <div className="lg:col-span-1">...</div>
</div>
```

**Purpose:** Two-column layout (main content + sidebar)

**How it works:**
- `grid-cols-1` - 1 column on mobile
- `lg:grid-cols-3` - 3 columns on large screens
- Left content spans 2 columns (`lg:col-span-2`)
- Right sidebar spans 1 column (`lg:col-span-1`)
- `gap-8` - Spacing between columns

**Result:** 
- Mobile: Stack vertically
- Desktop: 2/3 width for content, 1/3 for sidebar

</Accordion>

<Accordion title="Sticky Sidebar">

```jsx
<div className="sticky top-4">
  {/* Booking card content */}
</div>
```

**Purpose:** Keep booking card visible while scrolling

**How it works:**
- `sticky` - Stick to viewport when scrolling
- `top-4` - Stay 1rem (16px) from top

The booking card stays in view as users scroll through the listing details!

## Page Component Best Practices

<Accordions>

<Accordion title="1. Keep Pages Focused">

**Each page should have one primary purpose:**

```jsx
// ✅ Good - Focused on listing details
function ListingDetailsPage() {
  return (
    <div>
      <ListingInfo />
      <BookingCard />
    </div>
  );
}

// ❌ Bad - Too many responsibilities
function ListingDetailsPage() {
  return (
    <div>
      <Navigation />
      <ListingInfo />
      <SearchFilters />
      <BookingCard />
      <Footer />
      <ChatWidget />
    </div>
  );
}
```

</Accordion>

<Accordion title="2. Extract Complex Sections">

**Move complex UI into separate components:**

```jsx
// ✅ Good - Extracted components
function ListingDetailsPage() {
  return (
    <div>
      <ListingHeader listing={listing} />
      <ImageGallery images={images} />
      <ListingContent listing={listing} />
      <BookingCard listing={listing} />
    </div>
  );
}

// ❌ Bad - Everything inline
function ListingDetailsPage() {
  return (
    <div>
      {/* 500 lines of JSX... */}
    </div>
  );
}
```

We'll do this extraction in Lesson 8!

</Accordion>

<Accordion title="3. Handle Loading and Errors">

**Pages should manage their own states:**

```jsx
function ListingDetailsPage() {
  const [listing, setListing] = useState(null);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState(null);

  if (isLoading) return <Spinner />;
  if (error) return <ErrorMessage error={error} />;
  if (!listing) return <NotFound />;

  return <div>{/* Content */}</div>;
}
```

We'll implement this in Lesson 7!

</Accordion>

<Accordion title="4. Use Semantic HTML">

**Structure pages with proper HTML5 elements:**

```jsx
// ✅ Good - Semantic structure
<main className="container">
  <header>
    <h1>Title</h1>
  </header>
  
  <article>
    <section>Content</section>
  </article>
  
  <aside>
    <div>Sidebar</div>
  </aside>
</main>

// ❌ Bad - All divs
<div className="container">
  <div>
    <div>Title</div>
  </div>
  <div>Content</div>
</div>
```

</Accordion>
