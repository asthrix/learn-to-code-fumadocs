# Environment Variables

## Learning Objectives

By the end of this lesson, you will:

-  [ ] Create environment files to keep secrets out of source control.
-  [ ] Reference environment variables inside Vite/React code.
-  [ ] Share a safe `.env.example` template with the team.
-  [ ] Adopt practices for rotating keys and documenting usage.

## Project Context

TaskFlow Pro consumes API URLs, analytics keys, and feature flags. Storing them
in `.env` files keeps the repo safe and makes deployments reproducible.

---

## Environment Files

Create the following files at the project root:

-  `.env.example` ‚Üí public template checked into Git.
-  `.env.local` ‚Üí developer-specific values ignored by Git.

### Basic Example

```
# .env.example
VITE_API_URL=http://localhost:4000/api
VITE_APP_NAME=TaskFlow Pro
```

### Practical Example

```ts
const api = axios.create({ baseURL: import.meta.env.VITE_API_URL });
```

---

## ‚úÖ Best Practices

### 1. Prefix with `VITE_`

**Why:** Vite only exposes variables starting with `VITE_` to the client bundle.

```
VITE_FEATURE_UPCOMING_TASKS=true
```

### 2. Document Usage

**Why:** Clarifies what each value controls and when to rotate it.

```md
-  `VITE_API_URL`: Base endpoint for REST API. Swap to production URL in M8.
```

---

## ‚ùå Common Mistakes

### 1. Committing Secrets

**Problem:** Sensitive keys end up in Git history.

```bash
# Bad: adding `.env.local` to commits.
git add .env.local
```

**Solution:** `.gitignore` should include `.env*` excluding `.env.example`.

### 2. Missing Fallbacks

**Problem:** App crashes when the variable is undefined.

```ts
// Bad
const apiUrl = import.meta.env.VITE_API_URL;
```

**Solution:**

```ts
const apiUrl = import.meta.env.VITE_API_URL ?? "http://localhost:4000/api";
```

## üî® Implement in TaskFlow Pro

### Task: Configure Env Files

1. Add `.env.example` with placeholder values for API URL, product name, and
   analytics ID (future use).
2. Create `.env.local` with your local API URL.
3. Update `.gitignore` to exclude `.env*` except `.env.example`.
4. Reference `import.meta.env.VITE_API_URL` in a sample util (e.g.,
   `src/lib/api.ts`).

#### Expected Result

Running `npm run dev` reads from `.env.local`, while teammates can copy values
from `.env.example`.
