## Component Inventory

| Component  | Purpose                           | Key Props                     |
| ---------- | --------------------------------- | ----------------------------- |
| `Header`   | Global navigation + quick actions | `user`, `onToggleSidebar`     |
| `Sidebar`  | Project/Filter navigation         | `projects`, `activeProjectId` |
| `TaskCard` | Display single task               | `task`, `onStatusChange`      |

### Basic Example

```tsx
type TaskCardProps = {
   title: string;
   priority: "low" | "medium" | "high";
};
```

### Practical Example

```tsx
export function TaskCard({ task, onStatusChange }: TaskCardProps) {
   return (
      <article className='rounded-lg border bg-card p-4 shadow-sm'>
         <header className='flex items-center justify-between'>
            <h3 className='text-base font-semibold'>{task.title}</h3>
            <span className={`badge badge-${task.priority}`}>
               {task.priority}
            </span>
         </header>
         <p className='mt-2 text-sm text-muted-foreground'>
            {task.description}
         </p>
         <button
            type='button'
            className='mt-3 text-sm font-medium text-primary hover:text-primary/80'
            onClick={() => onStatusChange(task.id)}
         >
            Mark as {task.status === "done" ? "To-do" : "Done"}
         </button>
      </article>
   );
}
```

## Tailwind Tokens

Create a `tailwind.config.ts` file that exposes design tokens such as colors
(`fd-primary`), spacing scale, and typography. Components should consume these
tokens rather than hard-coded values.

## ‚ùå Common Mistakes

### 1. Ignoring Accessibility

**Problem:** Buttons without accessible names or focus states hinder users.

```tsx
// Bad: clickable span without role or aria-label.
<span onClick={...}>More</span>
````

**Solution:**

```tsx
<button type='button' aria-label='View task details'>
   More
</button>
```

### 2. Styling with Inline Magic Numbers

**Problem:** Hard-coded values drift from design tokens over time.

```tsx
// Bad
<div style={{ padding: "13px 27px" }} />
```

**Solution:**

```tsx
<div className='px-4 py-3' />
```
