# Implementing Module 0

## Learning Objectives

By the end of this lesson, you will:

-  [ ] Bootstrap the TaskFlow Pro app from scratch using Vite.
-  [ ] Set up Tailwind CSS with custom theme tokens.
-  [ ] Add a base layout with placeholder Header and Sidebar regions.
-  [ ] Run lint/format scripts to confirm configuration.

## Project Context

This is the first hands-on build lesson. You will produce the repository
foundation that later modules extend.

---

## Step 1: Generate the Project

```bash
npm create vite@latest taskflow-pro -- --template react-ts
cd taskflow-pro
npm install
```

Clean up boilerplate files (`src/assets`, `src/App.css`, etc.) to prepare for
Tailwind.

---

## Step 2: Configure Tailwind

```bash
npx tailwindcss init -p
```

Update `tailwind.config.js`:

```js
module.exports = {
   content: ["./index.html", "./src/**/*.{ts,tsx}"],
   theme: {
      extend: {
         colors: {
            "fd-primary": "#2563eb",
            "fd-primary-foreground": "#ffffff",
            "fd-accent": "#e0f2fe",
         },
      },
   },
};
```

Replace `src/index.css` with Tailwind directives (`@tailwind base;`, etc.).

---

## ‚úÖ Best Practices

### 1. Remove Unused Assets Immediately

**Why:** Keeps git history clean and avoids unused code warnings.

```bash
rm -rf src/assets src/App.css src/index.css
```

### 2. Commit After Each Major Step

**Why:** Easier to revert if configuration needs adjusting.

```bash
git commit -am "chore: scaffold taskflow pro"
```

---

## ‚ùå Common Mistakes

### 1. Forgetting Tailwind Directives

**Problem:** Styles do not load.

```css
/* Bad: missing Tailwind directives */
```

**Solution:**

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

### 2. Skipping Lint Run

**Problem:** Configuration issues surface later when harder to debug.

```bash
# Bad: ignoring lint output after initial setup.
```

**Solution:**

```bash
npm run lint
```
