# Setup and Requirements

## Learning Objectives

By the end of this lesson, you will:

-  [ ] Have Node.js and npm installed on your system
-  [ ] Have VS Code configured with React development extensions
-  [ ] Have Git installed and configured for version control
-  [ ] Be able to verify all tools are working correctly

## Project Context

Before we can build TaskFlow Pro, we need to set up the proper development
tools. This is a **critical foundation** - taking time to set this up correctly
will save hours of troubleshooting later.

## System Requirements

### Minimum Hardware

-  **Processor:** Dual-core CPU (Intel Core i3 or equivalent)
-  **RAM:** 4GB minimum, 8GB+ recommended
-  **Storage:** 5GB free space for Node.js, tools, and project files
-  **Operating System:** Windows 10+, macOS 10.12+, or Linux (Ubuntu 18.04+)

### Internet Connection

-  Required for downloading tools and packages
-  Recommended for accessing documentation
-  Needed for deploying your application later

## Step 1: Install Node.js and npm

Node.js is the JavaScript runtime that allows you to run JavaScript outside the
browser. npm (Node Package Manager) comes bundled with Node.js and manages
project dependencies.

### Why Do We Need Node.js?

-  **Run build tools** like Vite for development
-  **Install packages** from the npm registry
-  **Run scripts** for development tasks
-  **Bundle and optimize** code for production

### Installation Steps

#### For Windows:

1. **Visit the official website**

   ```
   https://nodejs.org
   ```

2. **Download the LTS version** (Long Term Support)

   -  Click the green "LTS" button (e.g., "18.17.0 LTS")
   -  LTS versions are more stable for production use

3. **Run the installer**

   -  Double-click the downloaded `.msi` file
   -  Follow the installation wizard
   -  ✅ Check "Automatically install necessary tools"
   -  ✅ Accept the license agreement
   -  Use default installation location

4. **Complete installation**
   -  Click "Install"
   -  Wait for the process to complete
   -  Click "Finish"

#### For macOS:

**Option 1: Official Installer**

1. Visit [https://nodejs.org](https://nodejs.org)
2. Download the LTS `.pkg` file
3. Run the installer and follow prompts
4. Enter your password when prompted

**Option 2: Using Homebrew** (Recommended)

```bash
# Install Homebrew if you don't have it
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Node.js
brew install node

# Verify installation
node --version
npm --version
```

#### For Linux (Ubuntu/Debian):

```bash
# Update package index
sudo apt update

# Install Node.js and npm
sudo apt install nodejs npm

# Verify installation
node --version
npm --version

# If version is too old, use NodeSource repository:
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Verify Installation

Open your **terminal** (Command Prompt on Windows, Terminal on macOS/Linux) and
run:

```bash
# Check Node.js version
node --version
# Should show: v18.x.x or higher

# Check npm version
npm --version
# Should show: 9.x.x or higher
```

**Expected Output:**

```
$ node --version
v18.17.0

$ npm --version
9.6.7
```

## Step 2: Install Visual Studio Code

VS Code is a powerful, free code editor perfect for React development.

### Why VS Code?

-  **Free and open-source**
-  **Excellent React support** with extensions
-  **Integrated terminal** for running commands
-  **IntelliSense** for code completion
-  **Built-in Git integration**
-  **Large extension marketplace**

### Installation Steps

1. **Download VS Code**

   ```
   https://code.visualstudio.com
   ```

2. **Install for your platform**

   -  **Windows:** Run the `.exe` installer
   -  **macOS:** Drag to Applications folder
   -  **Linux:** Install `.deb` or `.rpm` package

3. **Launch VS Code**
   -  Open the application
   -  Accept any initial setup prompts

### Essential VS Code Extensions

Install these extensions for optimal React development:

#### 1. ES7+ React/Redux/React-Native snippets

**Install:**

1. Click Extensions icon (or press `Ctrl+Shift+X` / `Cmd+Shift+X`)
2. Search for "ES7+ React"
3. Click "Install" on "ES7+ React/Redux/React-Native snippets" by dsznajder

**Features:**

-  Quick React component snippets
-  JSX syntax shortcuts
-  Import helpers

#### 2. Prettier - Code Formatter

**Install:**

1. Search "Prettier - Code formatter"
2. Install by Prettier

**Features:**

-  Automatic code formatting
-  Consistent style across team
-  Format on save

**Configuration:**

```json
// Settings.json (File → Preferences → Settings → Open Settings JSON)
{
   "editor.formatOnSave": true,
   "editor.defaultFormatter": "esbenp.prettier-vscode",
   "prettier.singleQuote": true,
   "prettier.semi": true,
   "prettier.tabWidth": 2
}
```

#### 3. ESLint

**Install:**

1. Search "ESLint"
2. Install by Microsoft

**Features:**

-  Catch errors before runtime
-  Enforce coding standards
-  Show warnings and errors inline

#### 4. Auto Rename Tag

**Install:**

1. Search "Auto Rename Tag"
2. Install by Jun Han

**Features:**

-  Automatically rename paired HTML/JSX tags
-  Saves time and prevents errors

#### 5. Auto Close Tag

**Install:**

1. Search "Auto Close Tag"
2. Install by Jun Han

**Features:**

-  Automatically add closing tags
-  Works with JSX elements

#### 6. Tailwind CSS IntelliSense

**Install:**

1. Search "Tailwind CSS IntelliSense"
2. Install by Tailwind Labs

**Features:**

-  Autocomplete Tailwind classes
-  Preview colors and styles
-  Linting for Tailwind CSS

#### 7. Path Intellisense

**Install:**

1. Search "Path Intellisense"
2. Install by Christian Kohler

**Features:**

-  Autocomplete file paths
-  Faster imports

### Recommended Settings

Add these to your VS Code settings:

```json
{
   "editor.formatOnSave": true,
   "editor.codeActionsOnSave": {
      "source.fixAll.eslint": true
   },
   "editor.tabSize": 2,
   "editor.wordWrap": "on",
   "files.autoSave": "afterDelay",
   "files.autoSaveDelay": 1000,
   "emmet.includeLanguages": {
      "javascript": "javascriptreact"
   },
   "emmet.triggerExpansionOnTab": true
}
```

## Step 3: Install Git

Git is the industry-standard version control system. You'll use it to track
changes and collaborate.

### Why Git?

-  **Track changes** to your code over time
-  **Collaborate** with other developers
-  **Backup** your work remotely
-  **Deploy** applications with Git-based platforms

### Installation Steps

#### For Windows:

1. **Download Git**

   ```
   https://git-scm.com/download/win
   ```

2. **Run installer**
   -  Accept defaults for most options
   -  ✅ Select "Use Git from the Windows Command Prompt"
   -  ✅ Select "Checkout Windows-style, commit Unix-style line endings"

#### For macOS:

**Option 1: Xcode Command Line Tools**

```bash
xcode-select --install
```

**Option 2: Homebrew**

```bash
brew install git
```

#### For Linux:

```bash
# Ubuntu/Debian
sudo apt install git

# Fedora
sudo dnf install git
```

### Verify Git Installation

```bash
git --version
# Should show: git version 2.x.x or higher
```

### Configure Git

Set your identity (required for commits):

```bash
# Set your name
git config --global user.name "Your Name"

# Set your email
git config --global user.email "your.email@example.com"

# Verify configuration
git config --list
```

## ✅ Best Practices

### 1. Keep Tools Updated

**Why:** Security fixes, bug patches, and new features

```bash
# Update npm itself
npm install -g npm@latest

# Check for outdated global packages
npm outdated -g
```

### 2. Use LTS Versions

**Why:** Long Term Support versions are stable and well-tested

-  **Good:** Node.js 18.x.x LTS
-  **Avoid:** Odd-numbered versions (17.x, 19.x) - these are cutting-edge but
   less stable

### 3. Organize Your Projects

**Why:** Easy to find and manage your work

```
Developer/
├── React-Course/          # Course projects
├── Personal-Projects/     # Side projects
└── Work/                  # Professional work
```

## ❌ Common Mistakes

### 1. Not Using Administrator/Sudo Rights

**Problem:** Installation fails due to permissions

```bash
# Bad: Permission denied errors
npm install -g some-package

# Good: Use appropriate privileges
# Windows: Run terminal as Administrator
# macOS/Linux: Use sudo when needed
sudo npm install -g some-package
```

### 2. Installing Too Many Global Packages

**Problem:** Version conflicts and system bloat

```bash
# Bad: Installing everything globally
npm install -g create-react-app webpack babel eslint prettier

# Good: Use npx for one-time commands
npx create-react-app my-app

# Good: Install project-specific tools locally
npm install --save-dev eslint prettier
```

### 3. Ignoring Path Issues

**Problem:** Commands not found after installation

**Solution (Windows):**

-  Search "Environment Variables" in Start menu
-  Edit PATH variable
-  Add Node.js installation directory
-  Restart terminal

**Solution (macOS/Linux):**

```bash
# Add to ~/.bash_profile or ~/.zshrc
export PATH="$PATH:/usr/local/bin"

# Reload shell
source ~/.bash_profile
```

## Troubleshooting

### Node.js Command Not Found

**Symptoms:** `node: command not found`

**Solutions:**

1. Restart terminal after installation
2. Check PATH environment variable
3. Reinstall Node.js with option to add to PATH

### npm Permission Errors

**Symptoms:** `EACCES` or permission denied errors

**Solutions:**

**macOS/Linux:**

```bash
# Change npm default directory
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bash_profile
source ~/.bash_profile
```

**Windows:**

-  Run terminal as Administrator
-  Or change npm default directory

### VS Code Extensions Not Working

**Symptoms:** Extensions installed but not functioning

**Solutions:**

1. Reload VS Code: `Ctrl+Shift+P` → "Reload Window"
2. Check extension conflicts
3. Update VS Code to latest version
4. Reinstall problematic extensions

### Port Already in Use

**Symptoms:** `Port 5173 is already in use`

**Solutions:**

```bash
# Find process using port (macOS/Linux)
lsof -i :5173
kill -9 <PID>

# Windows
netstat -ano | findstr :5173
taskkill /PID <PID> /F

# Or use different port
npm run dev -- --port 3000
```
