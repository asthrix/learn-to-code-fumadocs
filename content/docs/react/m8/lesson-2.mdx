## Deploy Your Application

Let's deploy! Make sure you're in your project directory.

<Steps>

<Step>

**Run deployment command:**

```bash title="Terminal"
vercel
```

This single command starts the entire deployment process! ‚ú®

</Step>

<Step>

**Answer configuration questions:**

Vercel will ask several questions on first deployment:

```bash
Vercel CLI 33.0.1
? Set up and deploy "~/projects/booking-app"? [Y/n]
```

**Type:** `Y` (Yes) and press Enter

```bash
? Which scope do you want to deploy to?
```

**Choose:** Your account name (should be selected by default)

```bash
? Link to existing project? [y/N]
```

**Type:** `N` (No - first time deploying)

```bash
? What's your project's name? (booking-app)
```

**Type:** Project name (or press Enter to use default)

**Tips for naming:**
- Use lowercase
- No spaces (use hyphens: `booking-app`)
- Descriptive but short
- This becomes part of your URL

```bash
? In which directory is your code located? ./
```

**Type:** `./` (current directory) or press Enter

</Step>

<Step>

**Vercel auto-detects framework:**

```bash
Auto-detected Project Settings (Vite):
- Build Command: vite build
- Development Command: vite --port $PORT
- Install Command: npm install
- Output Directory: dist
? Want to modify these settings? [y/N]
```

**Type:** `N` (No - defaults are perfect for Vite)

<Callout type="info">
**Auto-detection is smart!** Vercel recognizes Vite, Next.js, Create React App, and dozens of other frameworks automatically.
</Callout>

</Step>

<Step>

**Watch the build process:**

Vercel will now build your application:

```bash
üîó  Linked to your-account/booking-app (created .vercel)
üîç  Inspect: https://vercel.com/your-account/booking-app/[id]
‚úÖ  Production: https://booking-app.vercel.app [2s]

üìù  Deployed to production. Run `vercel --prod` to overwrite later.
```

**What just happened:**

1. **Linked**: Connected local project to Vercel
2. **Inspect**: Link to view build logs
3. **Production**: Your live URL!
4. **Time**: Build completed in ~2 seconds

</Step>

<Step>

**Visit your deployed application!**

Click the production URL or open it in your browser:

```
https://booking-app.vercel.app
```

**üéâ Congratulations! Your app is LIVE!** üéâ

</Step>

</Steps>

## Configure Environment Variables

If your app uses environment variables (like API keys), add them to Vercel.

<Steps>

<Step>

**Open Vercel dashboard:**

Visit: [vercel.com/dashboard](https://vercel.com/dashboard)

</Step>

<Step>

**Select your project:**

Click on your `booking-app` project.

</Step>

<Step>

**Go to Settings:**

Click "Settings" tab in the project navigation.

</Step>

<Step>

**Add environment variables:**

1. Click "Environment Variables" in left sidebar
2. Click "Add New"
3. Enter variable details:

```
Name:  VITE_API_BASE_URL
Value: https://api.example.com
Environments: ‚úì Production ‚úì Preview ‚úì Development
```

**Important for Vite:** Environment variables must start with `VITE_` to be exposed to your application.

**Common variables:**

```bash
VITE_API_BASE_URL=https://api.example.com
VITE_API_KEY=your_api_key_here
VITE_STRIPE_PUBLIC_KEY=pk_live_...
```

</Step>

<Step>

**Redeploy with new variables:**

Environment variables only apply to NEW deployments:

```bash title="Terminal"
vercel --prod
```

This redeploys with your new environment variables.

</Step>

</Steps>

<Callout type="warning">
**Security:** Never commit `.env` files to Git! Always use Vercel dashboard for production secrets.
</Callout>

<Callout type="info">
**Vite specific:** Variables must start with `VITE_` to be accessible in your code:

```jsx
// ‚úÖ Accessible
const apiUrl = import.meta.env.VITE_API_BASE_URL;

// ‚ùå Not accessible (no VITE_ prefix)
const secret = import.meta.env.SECRET_KEY;
```
</Callout>

## Troubleshooting Common Issues

<Accordions>

<Accordion title="Build succeeds locally but fails on Vercel">

**Possible causes:**

1. **Missing dependencies:**

   Check `package.json` has all required packages:

   ```bash
   npm install
   git add package.json package-lock.json
   git commit -m "Update dependencies"
   vercel --prod
   ```

2. **Environment variables:**

   Add missing variables in Vercel dashboard.

3. **Node version:**

   Specify in `package.json`:

   ```json
   {
     "engines": {
       "node": "18.x"
     }
   }
   ```

4. **Case-sensitive imports:**

   Filenames must match exactly (Vercel uses Linux):

   ```jsx
   // ‚ùå Fails
   import Button from './Button';  // File: button.jsx

   // ‚úÖ Works
   import Button from './button';
   ```

</Accordion>

<Accordion title="Application shows blank page">

**Check these:**

1. **Console errors:**

   Open browser DevTools (F12) and check Console tab.

2. **Incorrect base path:**

   If deploying to subdirectory, configure in `vite.config.js`:

   ```js
   export default {
     base: '/subdirectory/'
   }
   ```

3. **API calls failing:**

   Check Network tab for failed requests. Update API base URL.

4. **Missing environment variables:**

   Add them in Vercel dashboard and redeploy.

</Accordion>

<Accordion title="Changes not appearing after deploy">

**Solutions:**

1. **Hard refresh browser:**

   - Windows/Linux: `Ctrl + Shift + R`
   - macOS: `Cmd + Shift + R`

2. **Check deployment URL:**

   Make sure you're visiting the correct URL (production not preview).

3. **Wait for propagation:**

   CDN updates can take 30-60 seconds.

4. **Verify deployment:**

   Check dashboard to ensure deployment succeeded.

</Accordion>

<Accordion title="API requests failing (CORS errors)">

**Problem:** Your API rejects requests from Vercel domain

**Solution:** Update API CORS settings:

```js
// Backend API (Express example)
app.use(cors({
  origin: [
    'http://localhost:5173',           // Development
    'https://booking-app.vercel.app'   // Production
  ],
  credentials: true
}));
```

**For external APIs:**

Some APIs require domain whitelisting in their dashboard.

</Accordion>

<Accordion title="Environment variables not working">

**Checklist:**

1. **Vite prefix:**

   ```bash
   # ‚úÖ Correct
   VITE_API_URL=https://api.example.com

   # ‚ùå Won't work
   API_URL=https://api.example.com
   ```

2. **Redeploy after adding:**

   Variables only apply to new builds:

   ```bash
   vercel --prod
   ```

3. **Check environment:**

   Ensure variable is enabled for "Production".

4. **Access correctly:**

   ```jsx
   // ‚úÖ Correct
   import.meta.env.VITE_API_URL

   // ‚ùå Wrong
   process.env.VITE_API_URL
   ```

</Accordion>
