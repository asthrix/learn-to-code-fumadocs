```jsx
  <h1 className="brand-logo">ğŸ–ï¸ Holidaze</h1>
```

**Why important?**

1. **Recognition:** User knows they're in right place
2. **Trust:** Professional branding builds confidence
3. **Context:** Tagline reinforces value proposition

**Best practices:**

```jsx

// âŒ Bad: Generic or missing

```

</Accordion>

<Accordion title="Form Container (Primary Action)">

**Purpose:** Guide user to main action

```jsx
<div className="sign-in-form-container">
  <h2>Sign in to your account</h2>
  <p>Welcome back! Please enter your details.</p>
  
  {/* Form here */}
</div>
```

**Title considerations:**

```jsx
// âœ… Good options:
"Sign in to your account"     // Clear, direct
"Welcome back"                // Friendly
"Sign in to continue"         // Action-oriented

// âŒ Avoid:
"Login"                       // Too technical
"Authentication Required"     // Too formal
"Enter Credentials"           // Too robotic
```

**Subtitle purpose:**

- Sets friendly tone
- Provides encouragement
- Reduces form anxiety

</Accordion>

<Accordion title="Footer Section (Secondary Actions)">

**Purpose:** Offer alternatives without distraction

```jsx
<div className="sign-in-footer">
  <p>
    Don't have an account?{' '}
    <a href="/register">Sign up</a>
  </p>
</div>
```

**Why at bottom?**

1. **Hierarchy:** Sign in is primary action
2. **Don't distract:** New users see sign-in first
3. **Convention:** Users expect links at bottom

**Common patterns:**

```jsx
// Account creation
"Don't have an account? Sign up"

// Password recovery
"Forgot your password? Reset it"

// Help
"Need help? Contact support"

// Terms
"By signing in, you agree to our Terms"
```

**Elevation system:**


```css

  0 10px 10px -5px rgba(0, 0, 0, 0.04); /* Ambient shadow */
```

Dual shadows create more realistic depth!

**Visual effect:**
