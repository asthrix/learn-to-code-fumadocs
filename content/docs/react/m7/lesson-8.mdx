**Problem:** Form submits without validation

**Common causes:**

```jsx

  // ‚ùå No resolver - validation won't work
});

useForm({
  resolver: zodResolver(signInSchema),

// 2. Wrong handleSubmit usage

<form onSubmit={handleSubmit(onSubmit)}>

// 3. Missing register

// ‚úÖ Fix: Add register

```

## Accessibility Improvements

<Tabs items={['Labels & IDs', 'ARIA Attributes', 'Keyboard Navigation', 'Error Announcements']}>

<Tab value="Labels & IDs">

**Every input needs a label:**

```jsx
// ‚úÖ Good: Label with htmlFor
<label htmlFor="email">Email</label>
<input id="email" {...register('email')} />

// ‚ùå Bad: No connection
<label>Email</label>
<input {...register('email')} />
```

**Why this matters:**

```
With htmlFor:
  ‚Üí Click label ‚Üí Focuses input
  ‚Üí Screen reader announces: "Email, edit text"

Without htmlFor:
  ‚Üí Click label ‚Üí Nothing happens
  ‚Üí Screen reader announces: "Edit text" (no context)
```

**Label patterns:**

```jsx
// Pattern 1: Wrapping (implicit)
<label>
  Email
  <input {...register('email')} />
</label>

// Pattern 2: For attribute (explicit, recommended)
<label htmlFor="email">Email</label>
<input id="email" {...register('email')} />
```

</Tab>

<Tab value="ARIA Attributes">

**Describe errors to screen readers:**

```jsx
<input
  id="email"
  {...register('email')}
  aria-invalid={errors.email ? 'true' : 'false'}
  aria-describedby={errors.email ? 'email-error' : undefined}
/>

{errors.email && (
  <p id="email-error" className="form-error" role="alert">
    {errors.email.message}
  </p>
)}
```

**What each does:**

```jsx
aria-invalid="true"
// Tells screen reader: "This field has an error"

aria-describedby="email-error"
// Connects error message to input
// Screen reader announces: "Email, invalid, {error message}"

role="alert"
// Announces error immediately when it appears
```

**Button states:**

```jsx
<button
  type="submit"
  disabled={isSubmitting}
  aria-busy={isSubmitting}
>
  {isSubmitting ? 'Signing in...' : 'Sign In'}
</button>
```

`aria-busy` tells screen readers: "This button is busy"

</Tab>
