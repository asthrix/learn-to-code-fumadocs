# Module 2: React Components Deep Dive

In this module, we'll explore React components in depth, learn about props,
composition patterns, and how to build more complex UIs.

## Component Types

React supports two main types of components:

### 1. Function Components (Recommended)

```jsx
// Function Component
function Welcome(props) {
   return <h1>Hello, {props.name}!</h1>;
}

// Arrow Function Component
const Welcome = (props) => {
   return <h1>Hello, {props.name}!</h1>;
};

// Simplified Arrow Function
const Welcome = ({ name }) => <h1>Hello, {name}!</h1>;
```

### 2. Class Components (Legacy)

```jsx
import React, { Component } from "react";

class Welcome extends Component {
   render() {
      return <h1>Hello, {this.props.name}!</h1>;
   }
}
```

**Note**: We'll focus on function components as they're the modern standard.

## Props in Detail

Props (properties) are how we pass data to components. They are read-only and
help make components reusable.

### Props Destructuring

```jsx
// Without destructuring
function UserCard(props) {
   return (
      <div>
         <h3>{props.user.name}</h3>
         <p>{props.user.email}</p>
         <p>{props.user.role}</p>
      </div>
   );
}

// With destructuring
function UserCard({ user }) {
   return (
      <div>
         <h3>{user.name}</h3>
         <p>{user.email}</p>
         <p>{user.role}</p>
      </div>
   );
}

// Nested destructuring
function UserCard({ user: { name, email, role } }) {
   return (
      <div>
         <h3>{name}</h3>
         <p>{email}</p>
         <p>{role}</p>
      </div>
   );
}
```

## Conditional Rendering

React allows you to conditionally render components or elements:

### Using If-Else

```jsx
function WelcomeMessage({ user }) {
   if (user) {
      return <h1>Welcome back, {user.name}!</h1>;
   } else {
      return <h1>Please sign in.</h1>;
   }
}
```

### Using Ternary Operator

```jsx
function WelcomeMessage({ user }) {
   return (
      <div>
         {user ? <h1>Welcome back, {user.name}!</h1> : <h1>Please sign in.</h1>}
      </div>
   );
}
```

### Using Logical AND (&&)

```jsx
function Notifications({ notifications }) {
   return (
      <div>
         <h2>Dashboard</h2>
         {notifications.length > 0 && (
            <div className='notifications'>
               <h3>You have {notifications.length} new notifications</h3>
               <ul>
                  {notifications.map((notification, index) => (
                     <li key={index}>{notification.message}</li>
                  ))}
               </ul>
            </div>
         )}
      </div>
   );
}
```

## Key Concepts Mastered

1. **Function vs Class Components**: Modern React uses function components
2. **Props**: Data flow from parent to child components
3. **Destructuring**: Clean way to extract props
4. **Conditional Rendering**: Show/hide elements based on conditions
5. **List Rendering**: Display arrays of data
6. **Component Composition**: Building complex UIs from simple components
7. **Children Prop**: Flexible component content

## Best Practices

1. **Keep components small and focused**
2. **Use descriptive prop names**
3. **Always provide keys for list items**
4. **Prefer function components**
5. **Use composition over inheritance**
6. **Extract reusable components**
