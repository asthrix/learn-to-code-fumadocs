## Build the Page

### Assemble Sections

Ensure the hero, highlights, services placeholder, testimonials, and contact
anchor exist‚Äîeven if content is still draft. Consistent IDs ensure in-page
navigation works.

### Validate Structure

Run the W3C validator or a local HTML linter to catch mistakes before styling
begins.

### Basic Example

```html
<main id="main-content" tabindex="-1">
   <section id="hero">...</section>
   <section id="highlights">...</section>
   <section id="services">...</section>
   <section id="testimonials">...</section>
```

## ‚úÖ Best Practices

### 1. Keep Sections Modular

**Why:** Treat each section as a module so you can convert them into includes or
components later.

```html
<section id="services" data-section="services">
   <h2 id="services-title">Ways we can work together</h2>
   <p>...</p>
</section>
```

### 2. Capture Outstanding Tasks Inline

**Why:** Comments or data attributes help future modules know where to inject
content or styling.

```html
<section id="testimonials">
   <!-- TODO: Replace placeholder quote after client approval -->
   <blockquote>...</blockquote>
</section>
```

---

## ‚ùå Common Mistakes

### 1. Mixing Presentation with Structure

**Problem:** Adding inline styles or layout tables at this stage distracts from
semantic focus.

```html
<!-- Bad: inline styles controlling layout -->
<section style="display:flex"></section>
```

**Solution:**

```html
<section id="services">...</section>
```

### 2. Forgetting to Update Navigation After Adding Sections

**Problem:** New sections without corresponding navigation links break the
experience.

```html
<!-- Bad: navigation lacks link to testimonials -->
<li><a href="#highlights">Work</a></li>
<li><a href="#contact">Contact</a></li>
```

**Solution:**

```html
<li><a href="#testimonials">Proof</a></li>
```

Module 1.
