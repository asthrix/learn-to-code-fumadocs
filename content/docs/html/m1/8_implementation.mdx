# Module 1 Implementation

## Learning Objectives

By the end of this lesson, you will:

-  [ ] Combine header, navigation, main sections, asides, and footer into a
       cohesive layout.
-  [ ] Confirm semantic integrity using validation tooling.
-  [ ] Record outstanding content or tooling requirements.
-  [ ] Publish a clean commit representing Module 1 state.

## Project Context

Module 1 delivers a layout that mirrors how production teams ship HTML. You now
have a collection of sections ready for styling, interaction, and richer
content. Treat this as the semantic baseline before layering CSS and JS.

## ‚ùå Common Mistakes

### 1. Leaving TODO Comments Untracked

**Problem:** Comments with `TODO` but no tracking leads to forgotten work.

```html
<!-- TODO: add new testimonial -->
```

**Solution:**

```md
notes/retro-log.md ‚Üí Add backlog item referencing the section.
```

### 2. Forgetting to Update Accessibility Docs

**Problem:** Structural changes without documentation break alignment.

```md
// Bad: accessibility manifesto still mentions old nav structure.
```

**Solution:**

```md
Update docs/accessibility.md with the latest structural context.
```
