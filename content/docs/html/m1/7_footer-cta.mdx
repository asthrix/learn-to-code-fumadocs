# Footer Calls to Action

## Learning Objectives

By the end of this lesson, you will:

-  [ ] Build a footer that provides navigation, contact, and trust information.
-  [ ] Present legal and policy links accessibly.
-  [ ] Offer a secondary call-to-action or newsletter sign-up.
-  [ ] Plan how the footer scales to multi-page portfolios.

## Project Context

The footer is your final chance to convert. It should reiterate how to reach
you, surface key resources, and communicate trust signals (location,
availability, legal details). A thoughtful footer also encourages visitors to
explore more content.

---

## Footer Composition

### Contact Information

Include email, location, and preferred contact method. Use `<address>` for
physical location and contact details.

### Secondary Navigation

Provide links to resume, case studies, articles, or social profiles.

### Legal & Policies

Include privacy policy, accessibility statement, and copyright.

### Basic Example

```html
<footer role="contentinfo">
   <address>Avery Stone ¬∑ hello@averystone.design ¬∑ Remote (UTC-5)</address>
   <nav aria-label="Footer navigation">
      <ul>
         <li><a href="#highlights">Work</a></li>
         <li><a href="#services">Services</a></li>
      </ul>
   </nav>
</footer>
```

## ‚úÖ Best Practices

### 1. Use `<address>` for Contact Information

**Why:** `<address>` communicates contact details semantically to browsers and
assistive tech.

```html
<address>
   Avery Stone ¬∑ Remote ¬∑ <a href="mailto:hello@averystone.design">Email</a>
</address>
```

### 2. Group Legal Links with `<small>` or `<ul>`

**Why:** Grouping legal details clarifies their purpose and improves
readability.

```html
<section aria-label="Legal">
   <small>&copy; 2025 Avery Stone</small>
   <ul>
      <li><a href="/legal/privacy.html">Privacy</a></li>
   </ul>
</section>
```

## ‚ùå Common Mistakes

### 1. Duplicating Navigation Without Context

**Problem:** Footer navigation needs its own `aria-label` to distinguish it from
primary nav.

```html
<nav>
   <ul>
      <li><a href="#highlights">Work</a></li>
   </ul>
</nav>
```

**Solution:**

```html
<nav aria-label="Footer navigation">...</nav>
```

### 2. Omitting Form Labels

**Problem:** Newsletter forms without labels are inaccessible.

```html
<input type="email" placeholder="Email" />
```

**Solution:**

```html
<label for="newsletter-email">Email</label>
<input id="newsletter-email" type="email" />
```
