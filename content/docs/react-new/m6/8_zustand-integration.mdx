# Zustand Integration

## Learning Objectives

By the end of this lesson, you will:

-  [ ] Use Zustand for localised shared state while keeping Redux for entities.
-  [ ] Provide hooks that combine selectors from both systems.
-  [ ] Ensure compatibility with server components and suspense.
-  [ ] Capture guidance for future contributors.

## Coexisting with Redux

-  Use Zustand for UI/experience state (preferences, modals, keyboard
   shortcuts).
-  Keep Redux for normalized entities and complex async workflows.
-  Bridge both via custom hooks.

```tsx
export function useDashboardState() {
   const preferences = usePreferencesStore((state) => ({
      theme: state.theme,
      density: state.density,
   }));
   const tasks = useAppSelector((state) => selectTasksForDashboard(state));
   return { ...preferences, tasks };
}
```

---

## Server Components Compatibility

-  Zustand supports `use` in server components via `useStore` helpers.
-  For Next.js app router, create providers that hydrate client state.

```tsx
"use client";
import { usePreferencesStore } from "@/state/preferencesStore";

export function PreferencesHydrator({
   initialState,
}: {
   initialState: PreferencesSnapshot;
}) {
   const store = usePreferencesStore();
   useEffect(() => {
      store.setState(initialState, true);
   }, [initialState, store]);
   return null;
}
```
