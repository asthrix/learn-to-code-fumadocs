## Functional Components

A functional component is a JavaScript function returning JSX. It can accept
props and contain hooks.

### Basic Example

```tsx
function Button({ children }: { children: React.ReactNode }) {
   return (
      <button className='rounded bg-primary px-3 py-2 text-white'>
         {children}
      </button>
   );
}
```

### Practical Example

```tsx
type DashboardShellProps = {
   sidebar: React.ReactNode;
   header: React.ReactNode;
   children: React.ReactNode;
---

import { Accordions, Accordion } from '@/components/accordion';
};

export function DashboardShell({
   sidebar,
   header,
   children,
}: DashboardShellProps) {
   return (
      <div className='flex min-h-screen'>
         <aside className='hidden w-64 border-r border-border bg-muted/20 p-6 md:block'>
            {sidebar}
         </aside>
         <main className='flex-1 p-6'>
            <header className='mb-6'>{header}</header>
            {children}
         </main>
      </div>
   );
}
```

## Composition Patterns

-  **Container + Presentational:** Containers fetch data; presentational
   components render UI.
-  **Layout Components:** Provide consistent structure (e.g., `DashboardShell`).
-  **Slots:** Pass components as props for flexibility (like `sidebar`,
   `header`).

---

## ‚úÖ Best Practices

### 1. Single Responsibility

**Why:** Easier testing and reuse.

```tsx
function TaskListHeader({ total }: { total: number }) {
   return <h2 className='text-lg font-semibold'>Tasks ({total})</h2>;
}
```

### 2. File Organisation

**Why:** Clear paths accelerate navigation.

```text
src/components/
  layout/
    DashboardShell.tsx
  task/
    TaskCard.tsx
```

---

## ‚ùå Common Mistakes

### 1. Too Many Levels of Nesting

**Problem:** Hard to follow component tree.

**Solution:** Extract nested JSX into their own components.

### 2. Re-export Chaos

**Problem:** Barrel files can hide component sources.

**Solution:** Introduce index files when the module stabilises, not prematurely.
