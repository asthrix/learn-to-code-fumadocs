# Implementing Module 1

## Learning Objectives

By the end of this lesson, you will:

-  [ ] Assemble Header, Sidebar, and TaskCard components.
-  [ ] Compose them into the dashboard layout using props.

This is where TaskFlow Pro becomes tangible. You will ship the foundational UI
shell students and stakeholders will interact with.

```tsx
               type='button'

               className={buttonVariants({ variant: "primary" })}
               type='button'

               New task

         </div>
      </div>
   );
}
```

## Step 3: TaskCard

Create `src/components/task/TaskCard.tsx`:

```tsx
import type { Task } from "@/types/task";
import { Badge } from "@/components/ui/badge";

type TaskCardProps = {
   task: Task;
};

export function TaskCard({ task }: TaskCardProps) {
   return (
      <article className='rounded-lg border border-border bg-card p-4 shadow-sm'>
         <header className='flex items-center justify-between'>
            <h3 className='text-base font-semibold'>{task.title}</h3>
            <Badge variant={task.priority === "high" ? "danger" : "default"}>
               {task.priority}
            </Badge>
         </header>
         <p className='mt-2 text-sm text-muted-foreground'>
            {task.description}
         </p>
         <footer className='mt-4 flex items-center justify-between text-xs text-muted-foreground'>
            <span>Due {new Date(task.dueDate).toLocaleDateString()}</span>
            <span>{task.status === "done" ? "Completed" : "In progress"}</span>
         </footer>
      </article>
   );
}
```

Ensure `Badge` exists (create if needed using earlier example).

---

## Step 4: Compose in Dashboard

Update `App.tsx`:

```tsx
import { Header } from "@/components/layout/Header";
import { Sidebar } from "@/components/layout/Sidebar";
import { DashboardShell } from "@/components/layout/DashboardShell";
import { TaskCard } from "@/components/task/TaskCard";

const sampleTasks: Task[] = [
   {
      id: "tsk-1",
      title: "Set up analytics events",
      description: "Capture task creation and completion metrics.",
      dueDate: "2025-05-21",
      priority: "high",
      status: "in-progress",
   },
   // add a few more tasks
];

export function App() {
   return (
      <DashboardShell sidebar={<Sidebar />} header={<Header />}>
         <section className='grid gap-4 md:grid-cols-2 xl:grid-cols-3'>
            {sampleTasks.map((task) => (
               <TaskCard key={task.id} task={task} />
            ))}
         </section>
      </DashboardShell>
   );
}
```

Replace `sampleTasks` with real data sources in later modules.

## ‚úÖ Best Practices

### 1. Snapshot Components

**Why:** Take screenshots or notes to measure progress after styling changes.

### 2. Use Story-like Playgrounds

**Why:** Render components in isolation (sandbox) to test states.

---

## ‚ùå Common Mistakes

### 1. Recreating Inline Styles

**Problem:** Duplicate styles creep in.

**Solution:** Use shared components and tokens (Badge, button variants).

### 2. Hardcoding Real Data

**Problem:** Makes later API integration harder.

**Solution:** Keep fixtures minimal and obvious placeholders.
