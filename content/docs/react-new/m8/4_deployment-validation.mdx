# Deployment Validation

## Automated Checks

```bash
npm run lint
npm run test
npm run build
```

-  Run smoke Cypress specs against the Vercel preview URL.
-  Validate API health via `/api/status` endpoint.

---

## Manual QA Matrix

| Scenario                 | Owner   | Expected Outcome                           |
| ------------------------ | ------- | ------------------------------------------ |
| Login + dashboard access | QA      | Successful login, dashboard widgets render |
| Invite acceptance        | Product | Invite token redeemed, onboarding sequence |
| Form submission          | QA      | Task creation completes, analytics event   |
| Mobile responsive checks | Design  | Layout adapts to 375px width               |

---

## Post-Deploy Monitoring

-  Watch error rates for 30 minutes post release.
-  Confirm analytics ingestion in the dashboard.
-  Review Vercel function logs for anomalies.
-  Capture deploy summary in release journal.

---

## Retro & Next Steps

-  Schedule a five-minute retro within 24 hours.
-  Document lessons learned and follow-up tasks.
-  Update the runbook with any new procedures.

---

## ✅ Validation Checklist

-  [ ] Automated pipeline succeeded (lint/test/build).
-  [ ] Manual QA matrix signed off by owners.
-  [ ] Monitoring dashboards checked post-release.
-  [ ] Release retrospective documented.
