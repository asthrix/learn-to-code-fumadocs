## Understanding Components

In React, **components** are the building blocks of your application. Think of
them as custom HTML elements that you create.

### Why Components?

**Without components (traditional approach):**

```html
<div class="homepage">
   <h1>Find your next stay</h1>
   <div class="search">...</div>
   <div class="listings">...</div>
</div>
```

Everything in one HTML file - hard to maintain and reuse.

**With components (React approach):**

```jsx
<HomePage>
   <SearchBar />
   <ListingList />
</HomePage>
```

Each piece is self-contained, reusable, and easy to understand.

## Component Structure

React components are just JavaScript functions that return JSX:

```jsx title="Basic Component Structure"
function ComponentName() {
   return <div>{/* Your JSX here */}</div>;
}

export default ComponentName;
```

**2. Create `src/pages/HomePage.jsx`:**

```jsx title="src/pages/HomePage.jsx"
function HomePage() {
   return (
      <div className='home-page'>

      </div>
   );
}

export default HomePage;
```

## Code Breakdown

Let's understand every part of this component:

### 1. Function Declaration

```jsx
function HomePage() {
   // Component logic here
}
```

**Key points:**

-  Component names must start with a **capital letter** (HomePage, not homePage)
-  It's just a regular JavaScript function
-  Modern React uses functional components (not class components)

### 2. Return Statement

```jsx
return <div className='home-page'>{/* JSX content */}</div>;
```

**Key points:**

-  Components must return JSX
-  Wrap multiple elements in a parent (like `<div>`)
-  Use parentheses for multi-line returns

### 3. JSX Content

```jsx
<div className='home-page'>
   <h1>Find your next stay</h1>
   <p>Discover amazing places to book for your next adventure</p>
</div>
```

**Key points:**

-  JSX looks like HTML but is JavaScript
-  Use `className` instead of `class` (class is a JavaScript keyword)
-  Can include any valid HTML elements

### 4. Export Statement

```jsx
export default HomePage;
```

**Key points:**

-  Makes the component available to other files
-  `default` means it's the main export from this file
-  Other files can now `import HomePage from './pages/HomePage'`

## Understanding JSX

JSX (JavaScript XML) is a syntax extension that lets you write HTML-like code in
JavaScript.

### JSX vs HTML

<Tabs items={["JSX", "HTML"]}>
<Tab value="JSX">

```jsx
// React Component with JSX
function Button() {
   const handleClick = () => alert("Clicked!");

   return (
      <button className='btn-primary' onClick={handleClick}>
         Click me
      </button>
   );
}
```

</Tab>
  
<Tab value="HTML">

```html
<!-- Traditional HTML -->
<button class="btn-primary" onclick="alert('Clicked!')">Click me</button>
```

</Tab>
</Tabs>

### Key Differences

| Feature                | JSX                | HTML           |
| ---------------------- | ------------------ | -------------- |
| Class attribute        | `className`        | `class`        |
| Event handlers         | `onClick`          | `onclick`      |
| JavaScript expressions | `{variable}`       | Not supported  |
| Self-closing tags      | `<img />`          | `<img>`        |
| Style attribute        | `{{color: 'red'}}` | `"color: red"` |

### JSX Rules

1. **Single root element:**

```jsx
// ✅ Correct - single root
return (
  <div>
    <h1>Title</h1>
    <p>Content</p>
  </div>
);

// ❌ Wrong - multiple roots
return (
  <h1>Title</h1>
  <p>Content</p>
);
```

2. **Close all tags:**

```jsx
// ✅ Correct
<img src="photo.jpg" />
<br />

// ❌ Wrong
<img src="photo.jpg">
<br>
```

3. **Use camelCase for attributes:**

```jsx
// ✅ Correct
<div className="container" onClick={handleClick}>

// ❌ Wrong
<div class="container" onclick={handleClick}>
```

## Component Naming Conventions

Follow these naming conventions for React components:

### File Names

-  **PascalCase:** `HomePage.jsx`, `ListingCard.jsx`
-  Match the component name
-  Use `.jsx` extension for files containing JSX

### Component Names

-  **PascalCase:** `HomePage`, `UserProfile`, `ListingCard`
-  Descriptive and clear
-  Indicate what the component does

```jsx title="Good Examples"
// ✅ Good names
function HomePage() {}
function UserProfile() {}
function ListingCard() {}
function SearchBar() {}
```

```jsx title="Bad Examples"
// ❌ Bad names
function homePage() {} // Should be PascalCase
function UP() {} // Not descriptive
function Listing() {} // Too generic
function page1() {} // Not meaningful
```

## Common Mistakes

<Steps>
  <Step>
    ### Lowercase component name
    
    **Wrong:**
    ```jsx
    function homePage() { }
    ```
    
    **Correct:**
    ```jsx
    function HomePage() { }
    ```
    
    React components must start with a capital letter.
  </Step>
  
  <Step>
    ### Missing export
    
    **Wrong:**
    ```jsx
    function HomePage() {
      return <div>Content</div>;
    }
    // No export!
    ```
    
    **Correct:**
    ```jsx
    function HomePage() {
      return <div>Content</div>;
    }
    export default HomePage;
    ```
  </Step>
  
  <Step>
    ### Multiple root elements
    
    **Wrong:**
    ```jsx
    return (
      <h1>Title</h1>
      <p>Text</p>
    );
    ```
    
    **Correct:**
    ```jsx
    return (
      <div>
        <h1>Title</h1>
        <p>Text</p>
      </div>
    );
    ```
  </Step>
</Steps>
