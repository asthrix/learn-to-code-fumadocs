## JSX Fundamentals

JSX lets you write UI markup directly inside JavaScript using familiar HTML-like
syntax.

### Basic Example

```tsx
const heading = <h1 className='text-2xl'>TaskFlow Pro</h1>;
```

### Practical Example

```tsx
export function TaskSummary({
   total,
   completed,
}: {
   total: number;
   completed: number;
}) {
   return (
      <header className='flex items-center justify-between'>
         <div>
            <h2 className='text-lg font-semibold'>Today</h2>
            <p className='text-sm text-muted-foreground'>
               {completed} of {total} tasks complete
            </p>
         </div>
         <span className='rounded-full bg-primary/10 px-3 py-1 text-xs font-medium text-primary'>
            {Math.round((completed / Math.max(total, 1)) * 100)}%
         </span>
      </header>
   );
}
```

## Expressions & Attributes

-  Use curly braces `{}` to inject JavaScript.
-  Use `className` instead of `class`.
-  Use camelCase for event handlers (`onClick`).

## ‚úÖ Best Practices

### 1. Keep JSX Clean

**Why:** Readable markup makes debugging easier.

```tsx
return (
   <section className='space-y-4'>
      <TaskSummary total={tasks.length} completed={completedCount} />
      <TaskList tasks={tasks} />
   </section>
);
```

### 2. Use Fragments Instead of `<div>` Clutter

**Why:** Avoids extra nodes that affect styling.

```tsx
return (
   <>
      <Header />
      <Main />
   </>
);
```

## ‚ùå Common Mistakes

### 1. Using Statements Inside JSX

**Problem:** JSX only allows expressions.

```tsx
// Bad
{
   if (isLoading) {
      return <Spinner />;
   }
}
```

**Solution:** Use ternaries or helper functions.

```tsx
{
   isLoading ? <Spinner /> : <TaskList tasks={tasks} />;
}
```

### 2. Forgetting to Close Tags

**Problem:** Components fail to compile.

```tsx
// Bad
<TaskCard>
```

**Solution:** Always close tags.

```tsx
<TaskCard />
```
