Let's understand what React is, how it works, and why it's become the most

## What is React?

**React is a JavaScript library for building user interfaces.** Created by
Facebook (now Meta) in 2011 and open-sourced in 2013, React has revolutionized
how we build web applications.

### Key Characteristics

**1. Declarative** You describe what the UI should look like, and React figures
out how to update it.

```jsx title="Example: Declarative vs Imperative"
// Imperative (without React)
const button = document.createElement('button');
button.textContent = 'Click me';
button.addEventListener('click', handleClick);
document.body.appendChild(button);

// Declarative (with React)

<button onClick={handleClick}>Click me</button>
```

**2. Component-Based** Build encapsulated components that manage their own
state, then compose them to create complex UIs.

```jsx title="Example: Component Composition"
// Small, reusable components
function Button({ children, onClick }) {
  return <button onClick={onClick}>{children}</button>;
}

function Card({ title, content }) { return ( <div> <h2>{title}</h2>

<p>{content}</p> </div> ); }

// Compose them together function App() { return (

<Card title='Welcome' content='Hello React!'>
   {" "}
   <Button onClick={() => alert("Clicked!")}> Click me </Button>{" "}
</Card> ); }
```

**3. Learn Once, Write Anywhere**
Use React for web (React DOM), mobile (React Native), desktop, VR, and more.

## How React Works

<Steps>
  <Step>
    ### Virtual DOM
    React creates a lightweight copy of the real DOM in memory
  </Step>

  <Step>
    ### Change Detection
    When data changes, React updates the Virtual DOM
  </Step>

  <Step>
    ### Diffing Algorithm
    React compares the new Virtual DOM with the old one
  </Step>

  <Step>
    ### Efficient Updates
    React updates only what changed in the real DOM
  </Step>
</Steps>

### The Virtual DOM

```jsx
// 1. Initial render
<div>
  <h1>Counter: 0</h1>
  <button>Increment</button>
</div>

// 2. User clicks button, counter becomes 1

// 3. React creates new Virtual DOM
<div>
  <h1>Counter: 1</h1>  {/* Changed */}
  <button>Increment</button>  {/* Same */}
</div>

// 4. React compares and finds only <h1> changed
// 5. React updates ONLY the <h1> in real DOM
````


This makes React incredibly fast, even with complex UIs.

## Core Concepts

### 1. JSX - JavaScript XML

JSX is a syntax extension that lets you write HTML-like code in JavaScript.

```jsx
// This JSX:
const element = <h1>Hello, {name}!</h1>;

// Compiles to: const element = React.createElement( 'h1', null, 'Hello, ',
name, '!' );

````

**Why JSX?**
- Easier to read and write than `createElement` calls
- Familiar HTML-like syntax
- Full power of JavaScript expressions
- Tooling support (syntax highlighting, autocomplete)

### 2. Components

Components are reusable pieces of UI. Think of them as custom HTML elements.

```jsx
// Functional Component (Modern)
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

// Arrow Function Style
const Welcome = (props) => {
  return <h1>Hello, {props.name}</h1>;
};

// Using the component
<Welcome name="Sarah" />
````


### 3. Props

Props (properties) are how you pass data to components.

```jsx
// Define component that accepts props
function UserCard({ name, age, avatar }) {
  return (
    <div>
      <img src={avatar} alt={name} />
      <h2>{name}</h2>
      <p>Age: {age}</p>
    </div>
  );
}

// Use component with props

<UserCard 
  name="John Doe" 
  age={28} 
  avatar="/john.jpg" 
/>
```

Props flow **one way** - from parent to child (called "unidirectional data
flow").

### 4. State

State is data that can change over time. When state changes, React re-renders
the component.

```jsx title="State Example"
import { useState } from 'react';

function Counter() { // Declare state variable const [count, setCount] =
useState(0);

return ( <div> <p>Count: {count}</p> <button onClick={() => setCount(count +
1)}> Increment </button> </div> ); }

```

**Props vs State:**
- **Props**: Passed from parent, immutable (read-only)
- **State**: Managed within component, mutable (can change)

## Why React?

<Accordions>
  <Accordion title="1. Performance">
    React's Virtual DOM and efficient diffing algorithm make updates fast, even for complex UIs with thousands of components.
  </Accordion>

  <Accordion title="2. Reusability">
    Components can be reused across your app and even in different projects. Build once, use everywhere.
  </Accordion>

  <Accordion title="3. Maintainability">
    Component-based architecture makes code easier to understand, test, and maintain. Each component has a single responsibility.
  </Accordion>

## React Ecosystem

React is more than just a library - it's an entire ecosystem:

### Core Libraries
- **React** - The core library
- **React DOM** - For web applications
- **React Native** - For mobile apps

### Popular Tools
- **Vite** - Fast build tool
- **Next.js** - React framework for production
- **React Router** - Client-side routing
- **Redux Toolkit** - State management
- **React Hook Form** - Form handling

### Styling Solutions
- **Tailwind CSS** - Utility-first CSS
- **styled-components** - CSS-in-JS
- **CSS Modules** - Scoped CSS

### UI Libraries
- **Material-UI** - Google's Material Design
- **Chakra UI** - Accessible components
- **shadcn/ui** - Copy-paste components
