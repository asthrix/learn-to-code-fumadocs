## Why Vite?

-  ‚ö° **Instant server start** - No bundling required
-  üî• **Lightning-fast HMR** - Hot Module Replacement
-  üì¶ **Optimized builds** - Pre-configured production builds
-  üéØ **Modern by default** - ES modules, TypeScript support
-  üõ†Ô∏è **Simple configuration** - Works out of the box

### Step-by-Step Instructions

**1. Open your terminal** and navigate to your projects folder:

   ```bash cd ~/projects # or wherever you keep your projects ```

**2. Create a new Vite project:**

   ```bash npm create vite@latest booking-app ```

**3. Answer the prompts:**

<Tabs items={["Recommended", "Alternative"]}>
   <Tab value='Recommended'>
      ``` ? Select a framework: ‚Ä∫ React ? Select a variant: ‚Ä∫ JavaScript ```
      We'll use JavaScript for simplicity. You can use TypeScript later once
      you're comfortable with React.
   </Tab>

   <Tab value='Alternative'>
      ``` ? Select a framework: ‚Ä∫ React ? Select a variant: ‚Ä∫ TypeScript ```
      Choose TypeScript if you're already familiar with it. The course examples
      work with both.
   </Tab>
</Tabs>

**4. Navigate into your project:**

```bash
cd booking-app 
```

**5. Install dependencies:**

```bash
npm install 
```

This downloads all required packages. It might take a minute.

**6. Start the development server:**

```bash
npm run dev 
```

You should see output like:

```
  VITE v5.0.0  ready in 234 ms

  ‚ûú  Local:   http://localhost:5173/
  ‚ûú  Network: use --host to expose
  ‚ûú  press h + enter to show help
```

**7. Open in your browser:**

Visit `http://localhost:5173/` and you should see the Vite + React welcome
screen!

### Important Files Explained

<Steps>
  <Step>
    ### `index.html`
    The entry point of your application. Contains the root `<div>` where React renders.
    
    ```html
    <!DOCTYPE html>
    <html lang="en">
      <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Vite + React</title>
      </head>
      <body>
        <div id="root"></div>
        <script type="module" src="/src/main.jsx"></script>
      </body>
    </html>
    ```
  </Step>
  
  <Step>
    ### `src/main.jsx`
    Renders your React app into the DOM. This is where React takes control.
    
    ```jsx
    import React from 'react'
    import ReactDOM from 'react-dom/client'
    import App from './App.jsx'
    import './index.css'

    ReactDOM.createRoot(document.getElementById('root')).render(
      <React.StrictMode>
        <App />
      </React.StrictMode>,
    )
    ```

    **What's happening:**
    - `ReactDOM.createRoot()` - Creates a React root
    - `document.getElementById('root')` - Finds the root div
    - `.render()` - Renders the App component
    - `<React.StrictMode>` - Enables development checks

  </Step>
  
  <Step>
    ### `src/App.jsx`
    Your main component. This is where you'll start building.
    
    ```jsx
    import { useState } from 'react'
    import './App.css'

    function App() {
      const [count, setCount] = useState(0)

      return (
        <div className="App">
          <h1>Vite + React</h1>
          <button onClick={() => setCount(count + 1)}>
            count is {count}
          </button>
        </div>
      )
    }

    export default App
    ```

  </Step>
  
  <Step>
    ### `package.json`
    Defines your project dependencies and scripts.
    
    ```json
    {
      "name": "booking-app",
      "private": true,
      "version": "0.0.0",
      "type": "module",
      "scripts": {
        "dev": "vite",
        "build": "vite build",
        "preview": "vite preview"
      },
      "dependencies": {
        "react": "^18.2.0",
        "react-dom": "^18.2.0"
      },
      "devDependencies": {
        "@vitejs/plugin-react": "^4.2.1",
        "vite": "^5.0.0"
      }
    }
    ```
    
    **Scripts:**
    - `npm run dev` - Start development server
    - `npm run build` - Build for production
    - `npm run preview` - Preview production build
  </Step>
  
  <Step>
    ### `vite.config.js`
    Vite configuration file. Usually you don't need to touch this.
    
    ```javascript
    import { defineConfig } from 'vite'
    import react from '@vitejs/plugin-react'

    export default defineConfig({
      plugins: [react()],
    })
    ```

  </Step>
</Steps>

## Understanding the Flow

Here's how your React app works:

```
1. Browser loads index.html
         ‚Üì
2. HTML loads src/main.jsx
         ‚Üì
3. main.jsx imports App component
         ‚Üì
4. App component is rendered
         ‚Üì
5. React injects App into #root div
         ‚Üì
6. You see your application!
```

## Common Issues

<Steps>
  <Step>
    ### Port Already in Use

    **Error:** `Port 5173 is already in use`

    **Solution:**
    - Kill the process using that port, or
    - Vite will automatically use the next available port
    - Check the terminal for the actual URL
  </Step>

  <Step>
    ### npm Command Not Found

    **Error:** `command not found: npm`

    **Solution:**
    - Ensure Node.js is installed
    - Restart your terminal
    - Check with `node --version`
  </Step>

  <Step>
    ### Module Not Found

    **Error:** `Cannot find module 'react'`

    **Solution:**
    - Run `npm install` again
    - Delete `node_modules` and run `npm install`
    - Check you're in the correct directory
  </Step>
</Steps>
